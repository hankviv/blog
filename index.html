Git Page + hexo + tk免费域名 打造属于自己的博客

发表于 2017-04-10 | 分类于 工具

引言

对于大多数来说，接触博客都从新浪博客，csdn等第三方博客开始，第三方博客有很多优点，比如免维护，简单的后台操作等，但是没法做个性化模板等设置，一般人都会从博客平台到租服务器自己搭建服务器，最后到托管到github上。

GitHub Pages是什么？

GitHub Pages本用于介绍托管在GitHub的项目，由于他的空间免费稳定，用来做搭建一个博客再好不过。

hexo

一个基于nodejs的博客系统

tk域名

Tk域名，网络上最容易申请到的免费域名之一，以TK结尾。是南太平洋岛国托克劳的顶级域名。

开始

首先注册github，
新建一个版本库
启用Github Page

下载并安装GIt

安装完成后，使用

git -v 查看是否安装成功

下载并安装nodejs

根据自己电脑操作系统的位数到git官网下载相应的版本：

https://nodejs.org/en/download/

检测是否安装成功，输入指令

node -v

安装hexo

npm install -g hexo-cli

初始化hexo

hexo init hexo

切换目录 （你需要存放blog的文件夹）

cd hexo
npm hexo
hexo generate
本地测试
hexo server

在浏览器输入：http://localhost:4000/ 即可访问到我们搭建好的hexo站点。

修改全局配置文件_config.yml

type: git
repository: https://github.com/你的Github用户名/你的Github用户名.github.io.git
branch: master

repository属性改成你的刚才创建仓库git地址。

安装hexo-deployer-git插件

npm install hexo-deployer-git –save
部署到Github上

依次执行以下三条命令：

  • $ hexo clean #清除缓存 网页正常情况下可以忽略此条命令

  • $ hexo generator #生成静态页面至public目录

  • $ hexo deploy #将.deploy目录部署到GitHub

  • 执行hexo deploy命令之后，如果最后一行打印出如下信息则表示部署成功

INFO Deploy done: git
然后你再去访问你创建的Github pages地址，也就是：你的Github用户名.github.io,即上面本地测试时的样子。

hexo使用主题

Hexo 安装主题的方式非常简单，只需要将主题文件拷贝至站点目录的 themes 目录下， 然后修改下配置文件即可。

hexo官方主题下载地址：https://hexo.io/themes/，里面有多种多样的主题模板供大家选择。

这里推荐一款很火的主题：next，下面的配置也是以这个主题为例。如果你使用的是其他的主题，那么请你自己根据说明文档进行配置。

next主题的官网，配置文档：http://theme-next.iissnan.com/

下载主题

next下载地址：https://github.com/iissnan/hexo-theme-next

启用主题

theme: next #博客主题，默认是landscape

####详细配置请参考 ： http://theme-next.iissnan.com/

###发布博客

hexo new “postName” #新建文章

hexo new “我的第一篇博客”

然后输入命令行

hexo clean
hexo generate
hexo deploy

然后打开就可以看到了

Github使用指南

安装git  

####git官网下载

####下载对应版本的git工具。

##本地新建仓库（create repository）
本地仓库和远程github仓库暂时没有联系。是新建在本机的一个仓库。 可以理解为一个目录，目录内的文件都将被git管理起来。
在相应目录输入命令 ：

mkdir blog
cd blog
git init

现在就创建了一个本地仓库 blog,注意git目录里面不要有中文路径。
创建后，本地文件内，会有一个.git文件夹出现。
当然现在就可以使用git 增加 提交来上传文件到本地版本库了。

git add 文件名
git commit -m ‘描述’

##管理本地仓库

####查看commit历史记录
git log
commit ea34578d5496d7dd233c827ed32a8cd576c5ee85
Author: Michael Liao askxuefeng@gmail.com
Date: Tue Aug 20 14:53:12 2013 +0800

其中commit后面的表示版本号，git版本号用head表示。

退回历史版本

版本号可以写前几位就可以，例如上面的版本号就可以填写 ea34578就好了

git reset –hard 版本号

##远程仓库管理

####github创建一个远程仓库。
在远程github主页上创建一个远程仓库blog。选择默认操作。

####将本地仓库和远程仓库关联
在本地仓库路径输入命令：

git remote add origin git@github.com:hankviv/blog.git

远程仓库的名字是origin，Git默认叫法为origin，但是origin名字让大家一看就知道是远程仓库，现在就可以push文件到远程仓库了。

git push -u origin master

-u是为了将本地master分支和远程master分支关联起来，以后只要使用 git push origin master就可以了。
本地仓库会有一个默认的主分支master，命令意思是push master分支上的提交到远程仓库。

###注：如果提交不成功，一般git会有一个警告，告诉用户没有权限操作。因为服务器没有你本机的rsa key。解决办法：
1.首先设置Git的user name和email：

git config –global user.name “hank”
git config –global user.email “hankdev@qq.com”

2.生成SSH密钥

ssh-keygen -t rsa -C “your_email@example.com”

然后点击确认

clip < ~/.ssh/id_rsa.pub

此命令是将rsa拷贝到粘贴板。
然后去github上添加ssh，步骤进入主页，点击头像，选择setting，ssh and GPG Keys。
最后测试，输入命令：

ssh -T git@github.com

##分支管理

你的github上的仓库会有一个默认的分支master，本地仓库也有一个默认的分支master

####查看本地分支:

git branch

####查看所有分支：

git branch -a

####创建本地分支：

git branch dev(分支名)

####创建远程分支：
首先在本地创建一个分支，然后push到远程。就会在远程创建一个分支。

git push origin dev(新创建的分支名)

####创建本地关联origin/dev的分支：

git checkout dev origin/dev

####切换本地分支

git checkout master(分支名)

####删除本地分支

git branch -d dev(分支名)

####删除远程分支

git push origin :dev(远程分支名)

##流程开发：

git checkout dev

切换到dev分支进行开发
开发完代码之后，我们有两个选择:

####第一种：
第一个：如果功能开发完成了，可以合并主分支。

git checkout master

切换到主分支

git merge dev

把dev分支的更改和master合并

git push

提交主分支代码远程

git checkout dev

切换到dev远程分支

git push

提交dev分支到远程

####第二个：如果功能没有完成，可以直接推送

git push

提交到dev远程分支
